# LP改善ログ（Portfolio）

BtoBのクラウド支援サービスを想定し、LP改善施策を「一覧 → 詳細」で管理・共有できるログアプリとして制作しました。  
施策ごとに **仮説 → 実装内容 → 検証観点** を残すことで、改善の意思決定を再現できる状態を目指しています。

約2週間、毎日2〜4時間ほど確保して、まずは「最低限動く（MVP）」状態を優先して実装・公開しました。現在も改善を継続しています。

## 想定ユースケース（BtoB前提）
- 目的：資料請求 / お問い合わせのCV向上
- 主なKPI：CTAクリック率、フォーム到達率、フォーム完了率
- 想定課題：
	- 商材の信頼性・実績・導入後イメージが伝わらないと検討が進まない
	- フォームの負荷が高いと離脱しやすい
	- 不安解消（費用感、体制、セキュリティ等）が不足するとCVにつながりにくい

## Demo
- https://lp-log.vercel.app/

## Features
- 一覧ページでログが表示される
- 詳細ページで1件のログが表示される（`/logs/[id]`）
- ステータスで絞り込みできる（URLクエリで状態保持）
- 0件時の空状態表示
- Vercelにデプロイして公開URLで閲覧可能

## 改善施策例（仮説 → 施策 → 検証観点）
- フォーム項目削減
	- 仮説：入力項目が多いほど離脱が増え、フォーム完了率が下がる
	- 施策：必須項目の見直し、項目削減（将来的に入力補助も検討）
	- 検証観点：フォーム到達→完了のCVR、入力開始後の離脱率
- FV訴求の具体化
	- 仮説：価値訴求が抽象的だと「自分ごと化」されず、CTAクリックに至らない
	- 施策：提供価値・対象・ベネフィットが具体的に伝わる文言へ調整
	- 検証観点：CTAクリック率、FV滞在時間、スクロール率
- FAQ追加
	- 仮説：不安要素（費用感、体制、セキュリティ等）が解消されないとCVにつながりにくい
	- 施策：よくある質問を追加し、意思決定に必要な情報を補完
	- 検証観点：FAQ閲覧率、FAQ閲覧後のCVR、離脱率

※現状は「施策と検証観点を一元管理する仕組み」の構築を優先しています。今後、計測ツール導入（GA等）を想定しています。
